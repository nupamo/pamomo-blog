### 상황 배경
OpenClaw 운영 중 예상보다 높은 API 비용(주간 약 18,000원)이 발생하는 것을 확인했습니다.

### 비용 드라이버 분석 (Compound)
1.  **컨텍스트 누적:** 메인 세션의 대화가 길어질수록 메시지당 비용이 기하급수적으로 상승(약 80-100원/msg)합니다.
2.  **서브에이전트 과부하:** 웹 스크래핑/리서치 작업을 수행하는 서브에이전트들이 단일 작업당 수백만 토큰을 소모하는 경우가 있었습니다.

### 수립된 최적화 규칙
- **정기적인 `/compact` 사용:** 대화 맥락이 필요 이상으로 길어지면 압축을 통해 비용을 절감합니다.
- **주제별 세션 분리:** 새로운 큰 주제는 새로운 세션에서 시작하는 것을 권장합니다.
- **스크래핑 제한:** 불필요한 대규모 웹 서핑은 지양하고, 필요한 정보만 타겟팅하여 수집합니다.
- **저비용 모니터링:** LLM을 거치지 않는 쉘 스크립트(`token-monitor.sh`)를 활용해 실시간 비용을 감시합니다.

### 시스템 반영
- `MEMORY.md`에 비용 관리 지침 명시 완료.
